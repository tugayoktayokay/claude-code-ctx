# ctx — Claude Code context manager

## Amaç

Claude Code session'larının token kullanımını canlı izleyen, büyüdüğünde session'a özel `/compact` prompt'u üreten, `/snapshot` skill'ine memory yazabilen bir tool. Sıfır dependency, pure Node built-in.

## Kökeni

`/Users/tugayoktayokay/tools/npc/` altında **npc-context.js** (601 satır) ve **npc-compact.js** (620 satır) zaten solid bir temel. Bunların mantığı kopyalanacak, birleştirilecek, problematik parçalar ayıklanacak. **npc-daemon.js**'in file watch + git hook fikirleri kısmen alınacak (CLAUDE.md overwrite kuralları HARİÇ).

## Temel prensip

- **Sıfır hook** — Claude Code session'ına token yükü getirme. Hiçbir şey `.claude/settings.json` hook'u olarak çalışmaz. Kullanıcı ayrı terminalde çalıştırır.
- **JSONL-based** — tek veri kaynağı `~/.claude/projects/<encoded-cwd>/*.jsonl`. Claude Code'un kendi transcript'i.
- **Zero AI** — sadece matematik + regex + pattern matching.
- **Zero deps** — `require` sadece Node built-in: `fs`, `path`, `os`, `child_process`.
- **CLAUDE.md'ye asla yazma** — bu problematikti.

## Commands

```bash
ctx                              # One-shot analiz: mevcut session özeti + öneri
ctx watch                        # Canlı monitor, macOS notif, terminal status
ctx compact                      # /compact prompt üret + clipboard'a kopyala
ctx snapshot [--name NAME]       # Session state'i memory formatında yaz
ctx history [N]                  # Son N session'ın metrikleri
ctx config                       # Config dosyasının yolu ve mevcut değerler
ctx skills                       # (opsiyonel) Projeye uygun skill önerileri
ctx --help
```

### `ctx` (default)

Aktif session'ı analiz et, tek seferlik özet bas:
- Context bar (% + token + model-aware MAX)
- Son N kullanıcı mesajı
- Kategoriler (top 4)
- Değiştirilen dosyalar
- Öneri: "rahat / dikkat / compact / snapshot + clear"

### `ctx watch`

10sn aralıkla transcript'i kontrol, eşik geçilince uyar:
- Terminal: tek satır status (her tick)
- Eşik geçişlerinde: detaylı panel + macOS notification
- SIGINT'te temiz kapanış

### `ctx compact`

`npc-compact.js` mantığı:
- Session'ı kategorize et
- Kritik pattern'ları çıkar (karar, hata, endpoint, failed attempt)
- Tailored `/compact focus on X + koru: Y; Z + at: W + devam: "V"` üret
- macOS'ta clipboard'a kopyala, terminal'de göster

### `ctx snapshot` ⭐ YENİ — BU ENTEGRASYONUN KALBİ

`npc-compact.js`'in analiz motorunu + bizim `/snapshot` skill'inin memory formatını birleştir.

**Çıktı:**
```markdown
---
name: <auto-generated, session topic-based>
description: <top 3 category + last intent>
type: project
---
Devam eden iş (<ISO date>):
- Değiştirilen dosyalar: <list>
- Alınan kararlar: <from decisions>
- Açık sorular / failed: <from failedAttempts>
- Son görev: "<last user intent>"

**Why:** <category rationale>
**How to apply:** <stack + next steps>
```

**Nereye yazar:**
`~/.claude/projects/<encoded-cwd>/memory/project_<slug>_<timestamp>.md`
+ `MEMORY.md` index'ine append.

**Neden değerli:** Manuel `/snapshot` skill'imi çağırdığımda ben 1-2 dakika harcıyorum. `ctx snapshot` saniyeler içinde hazırlıyor → ben sadece review + commit ediyorum.

### `ctx history`

Tüm `~/.claude/projects/*/*.jsonl` dosyalarını tara, trend göster:
- Son 10 session
- Her biri için: mtime, context %, mesaj sayısı, urgency level
- Görsel: renkli liste

### `ctx config`

Config dosyasını göster. Yoksa default ile oluştur.

  "watch": {
    "interval_ms": 10000,
    "macos_notifications": true,

**Key feature:** eşikler `quality_ceiling` üzerinden hesaplanır, `max` değil. Opus 1M olsa bile 200k'da sarı alarm başlar — "teknik olarak sığar ama kalite düşer" gerçeği.

## Model detection

`src/models.js`:

```js
function detectModel(entries) {
  // JSONL'de her assistant entry'sinde entry.message.model var
  // En son kullanılanı döndür
  for (let i = entries.length - 1; i >= 0; i--) {
    const e = entries[i];
    if (e.type === 'assistant' && e.message?.model) return e.message.model;
  }
  return 'default';
}
```

Config'den ilgili model limit'ini çek. Model tanınmazsa `default`.

## Kritik pattern'lar (korunması gerekenler)

`npc-compact.js`'ten alınan `CRITICAL_PATTERNS`:
- `kararı?[:\s]` → karar
- `sonuç|result[:\s]` → sonuç
- `hata|error[:\s].{10,80}` → hata
- `çözüm|solution|fix[:\s]` → çözüm
- `TODO|FIXME|HACK|NOTE` → todo
- `migration.*(?:add|remove|alter)` → migration
- `port|url|endpoint[:\s]\S+` → endpoint
- `env|secret|key[:\s]\S+` → config
- `olmadı|çalışmadı|başarısız` → failed attempt

## Snapshot entegrasyonu (detaylı)

**Akış:**
1. Kullanıcı `ctx snapshot` çalıştırır
2. Son session JSONL'i bulunur
3. `analyzer.js` + `strategy.js` ile session özetlenir
4. `snapshot.js` memory-compatible markdown üretir
5. `~/.claude/projects/<slug>/memory/project_<auto-name>.md` yazılır
6. `MEMORY.md` index güncellenir (duplicate check ile)
7. Clipboard'a özet kopyalanır

**Auto-naming:** Top kategori + son intent'in ilk 30 karakteri. Slugify edilir.

Örnek: `project_api_auth_middleware_refactor.md`

**Why field otomasyonu:** Kategori label'larından türetilir.

**How to apply field:** Stack + son görev + open questions.
